# Dev Branch Auditas — 2026-05-19

Visiškas po-MVP audit'as. 12 issues su prioritetais. Skaityta: visi backend services, migrations, modeliai, frontend pages + komponentai, shared tipai.

## 1. Implementuota ne pilnai

### 1.1 Aprobacijos workflow — tik 1 žingsnio hardcoded

- **Failas**: `apps/api/src/services/requests.service.ts:40`
- **Problema**: `DEFAULT_WORKFLOW_LEVELS = ['AM_ADMIN']` hardcoded. Komentaras nurodo, kad AM-wide vėliau bus konfigūruojama per `approval_workflows` lentelę.
- **Trūksta**: `approval_workflows` lentelė + UI workflow konfigūracija AM admin'ams.

### 1.2 Plano konversija — DRAFT scenarijus weak

- **Failai**: `requests.service.ts:604-609`, `RequestWizard.tsx:87-89`
- **Problema**: DRAFT statuso plano konversija blokuojama backend'e, bet frontend'e nėra aiškios validacijos prieš submit.
- **Trūksta**: Unit testas `convertPlanToCurrentYear` su DRAFT input'u.

### 1.3 Biudžeto validacija — gali viršyti totalAmount

- **Failas**: `budgets.service.ts:120-199` (upsert action)
- **Problema**: Sistema leidžia sukurti biudžetą su 100 EUR `totalAmount` ir 200 EUR allocations. Dashboard'e likutis tampa neigiamas, bet vartotojas nemato warning'o.
- **Trūksta**: Validacija `totalAmount >= sum(allocations)`.

### 1.4 Attachment upload permissions — backend/frontend desync

- **Failai**: `requestAttachments.service.ts:67-79`, `AttachmentList.tsx:161`
- **Problema**: Backend grąžina 403 kai teikėjas bando įkelti `order_pdf`, bet frontend nepatikrina permissions prieš upload.
- **Trūksta**: Frontend helper'is `canUploadKind(attachment_type, user, request_status)`.

## 2. Edge case'ai ir bug rizikos

### 2.1 Ketvirtinės sumos vs bendra suma — float lyginimas

- **Failas**: `RequestWizard.tsx:233-236`
- **Problema**: `Math.abs(quarterlyDiff) > 0.01` — kombinuojant frontend formatavimo transformacijas ir backend normalize'ą, gali atsirasti tylios klaidos.
- **Trūksta**: Standartinis `normalizeDecimal(value)` helper'is tiek frontend'e, tiek validation'e.

### 2.2 Plan konversija — copy'na tuščius laukus

- **Failas**: `requests.service.ts:625-657`
- **Bug scenario**: Plano `year > currentYear`, `status = SUBMITTED`, `costDu = 0` — konversija sukuria tuščią DRAFT.
- **Trūksta**: `convertPlanToCurrentYear` privalo patikrinti, ar bent viena sum > 0.

### 2.3 Approval steps sequence — istorija susimaišo

- **Failas**: `requests.service.ts:526-539`
- **Problema**: Resubmit po RETURNED kuria naujus žingsnius sekvencu N+1. 10 resubmit'ų = 10 žingsnių serijų, vartotojas nemato, kurie kurioje iteracijoje.
- **Trūksta**: UI vizualus atskyrimas tarp serijų ("Round 1", "Round 2").

### 2.4 Dashboard per-tenant — neignoruoja prašymo metų

- **Failas**: `dashboard.service.ts:377-386`
- **Problema**: `totalApproved` per-tenant netikrinama, ar `r.year === year`. Patvirtinus planą iš 2027 m. einamais metais, suma įskaičiuojama į einamųjų metų statistiką.
- **Trūksta**: `r.year === year` patikrinimas per per-tenant breakdown.

### 2.5 Attachment deletion — RESTRICT vs UI klaidina

- **Failas**: `20260519140000_create_request_attachments.ts:27`
- **Problema**: `uploaded_by_user_id` turi `onDelete('RESTRICT')`. Vartotojo ištrynimas blokuojamas. Frontend rodo „Trinti", bet backend 403'uoja.
- **Trūksta**: Aiškesnis error message + frontend pre-check.

## 4. Backend kokybė — TS tipai ir error handling

### 4.1 `grantedAmount: { type: 'any' }`

- **Failas**: `requests.service.ts:669`
- **Problema**: Moleculer schema neapsaugo nuo invalid input'o. `NaN` patenka į DB.
- **Trūksta**: `type: 'string'` su regex arba `type: 'number'`.

### 4.2 Decimal sumų formatas — nestandartizuotas

- **Failai**: `requests.service.ts:148`, multiple
- **Trūksta**: Standartinis `toDecimalString(value, precision = 2)` helper'is.

### 4.3 Base64 validacija — weak

- **Failas**: `requestAttachments.service.ts:81-84`
- **Problema**: Regex `/^[A-Za-z0-9+/]+={0,2}$/` praleidžia broken base64.
- **Trūksta**: `Buffer.from(dataBase64, 'base64')` decode'inimas.

### 4.4 Decision action — nėra atominės transakcijos

- **Failas**: `requests.service.ts:730-795`
- **Problema**: Request + ApprovalStep + RequestComment atnaujinimas be `trx`. Partial failure → inkonsistentus state.
- **Trūksta**: `Request.knex().transaction()` wrap'inimas.
